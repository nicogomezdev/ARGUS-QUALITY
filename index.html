Calidad del software

Origen

Calidad

La palabra calidad tiene su raíz en el latín qualĭtas. Difundida por Marco Tulio Cicerón. Inicialmente definida como:
"La propiedad o conjunto de propiedades inherentes a algo, que permiten juzgar su valor."

Software

La palabra software es un anglicismo, no tiene traducción directa y precisa a español. Fue acuñada como el lado contrario de hardware en el siglo XX. Se define como:

  • Soft significa "blando" o "flexible"
  • Ware significa "mercancía" o "artículo"

¿Qué es Calidad de software?

IEEE Std. 610-1990, la describe como: "El grado con el que un sistema, componente o proceso cumple los requerimientos especificados y las necesidades o expectativas del cliente o usuario."

Normas

ISO 25001

Procesos de gestión de calidad

Establece la forma de planificar, medir y controlar la calidad del producto, durante su desarrollo y planificación.

ISO 9126

Modelo original de calidad de software

Clasifica en seis caracteristicas principales la calidad:

  • Funcionalidad
  • Fiabilidad
  • Usabilidad
  • Eficiencia
  • Mantenibilidad
  • Portabilidad

IEE 829

Contenido de la documentación de pruebas

Estandarizar la forma en que se planifican y registran las pruebas de software

ISO 25010

Modelo de calidad del software mas reciente

Define critorios de calidad del producto

ISO 12207

Ciclo de vida del software

Establece el paso a paso que debe seguirse durante la vida del software, desde su creación hasta el último uso

ISO 15504

Evaluación de procesos

Proprociona el modelo para evaluar en sus totalidad la madurez del software

ISO 9001

Getsión de calidad

Establece los requisitos para un sistema de gestión de calidad. No es dedicado unicamente a software.

ISO 27001

Seguridad de la información

Establece los parametros y las formas para gestionar la seguridad de la información.

ISO 29119

Pruebas de software

Es el estandar internacional para los procesos, documentación y técnicas asociadas a las pruebas de software. Asegura que las pruebas sean repetibles y efectivas cuantas veces sea aplicada

ISO 25000

Requisitos y evaluación de la calidad

Es el conjunto de metricas que sirven de guía para especificar y evaluar la calidad del software. Se enfoca en la medición y mejora de la calidad
